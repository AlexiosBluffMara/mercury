What it is

Mercury is a personal AI assistant. It runs on your own computer.

You talk to it from any of six places — your terminal, Discord, a web page, iMessage, email, your phone — and the same agent answers, with the same memory, on the same hardware. No round-trips to anyone else's cloud.

It comes with four specialist skill domains out of the box: Chicago college search, Chicago tax & tenant law, 3D web graphics development, and hackathon submission packaging — the last of which built this page.

The brain is Kimi K2.6, an open-weight model from Moonshot AI accessed through the Nous Portal. The body is Hermes Agent, an open-source framework from Nous Research.

Architecture

Six client surfaces fan into one Hermes Gateway → Kimi K2.6 brain → 4 custom skill domains → 5-source data layer → 3-node infra (Mac dev, 5090 inference, Cloud Run scale).

Run it yourself

Heads up: there is no public hosted version of the live agent. Mercury is local-first by design — the AI runs on your GPU, not ours. The architecture diagrams and demo video here are real, but to actually chat with the agent you need to clone, install, and bring your own hardware.

What you need: an NVIDIA GPU with 12 GB+ VRAM, Python 3.11+, Ollama, and ~30 GB of disk space.

# 1. Clone Mercury
git clone https://github.com/AlexiosBluffMara/mercury
cd mercury

# 2. Make a Python environment for it
python -m venv .venv
source .venv/bin/activate    # Windows: .venv\Scripts\activate

# 3. Install Mercury
pip install -e .

# 4. Pull the local AI models (~10 GB download)
ollama pull gemma4:e4b
ollama pull embeddinggemma:300m

# 5. Talk to it
mercury chat

That's it. Type a question. It thinks on your GPU, replies in your terminal.
