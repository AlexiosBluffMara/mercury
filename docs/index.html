What it is, in plain English

Mercury is a chat-based AI assistant. You talk to it from Discord, iMessage, your terminal, a web page, or email — and it does real work for you instead of just chatting back. It runs on your own computer, not someone else's cloud, so your conversations stay private.

You can teach it new tricks by dropping a markdown file into a folder. No coding required.

The brain that drives it is Kimi K2.6, a 2026 open-weight AI model that's especially good at writing code and following long instructions. The body it lives in is Hermes Agent, an open-source agent framework from Nous Research. Mercury is the customized version we built on top.

Architecture

Six client surfaces fan into one Hermes Gateway → Kimi K2.6 brain → 4 custom skill domains → 5-source data layer → 3-node infra (Mac dev, 5090 inference, Cloud Run scale).

Try it yourself

Heads up: there is no public hosted version of the live agent. Mercury is local-first by design — the AI runs on your GPU, not ours. The architecture diagrams and demo video here are real, but to actually chat with the agent you need to clone, install, and bring your own hardware.

What you need: an NVIDIA GPU with 12 GB+ VRAM, Python 3.11+, Ollama, and ~30 GB of disk space.

# 1. Clone Mercury
git clone https://github.com/AlexiosBluffMara/mercury
cd mercury

# 2. Make a Python environment for it
python -m venv .venv
source .venv/bin/activate    # Windows: .venv\Scripts\activate

# 3. Install Mercury
pip install -e .

# 4. Pull the local AI models (~10 GB download)
ollama pull gemma4:e4b
ollama pull embeddinggemma:300m

# 5. Talk to it
mercury chat

That's it. Type a question. It thinks on your GPU, replies in your terminal.
