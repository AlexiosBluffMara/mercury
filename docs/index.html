The Six-Door Office — how Mercury works

Picture a small office with one occupant: an assistant who already knows you. They've read your notes, they remember last week's conversation, they know which tools sit on your desk and which documents live in your filing cabinet. Most assistants in 2026 don't work this way — you walk into someone else's building, they get amnesia, and you start from scratch.

Mercury's office is in your building. The assistant lives on a single computer you own — by default, an RTX 5090 desktop sitting in Chicago. The desk has six doors:

▷
Terminal
Type to it from any shell, Mac or Windows.
▷
Discord
"Snowy The Bot" in any server you invite it to.
▷
Web
A private chat URL, reachable from any browser.
▷
iMessage
Text a relay number, replies arrive on your phone.
▷
Email
Send to a real address; the agent reads & replies.
▷
Mobile
Same chat, same memory, in your pocket.

Walk through any door, and you're talking to the same person. Tell it something on Discord at noon, ask about it from the terminal at four — it remembers. The brain is Gemma 4 E4B, an open-weight model from Google running locally at 194 tokens per second on the 5090. The body is Hermes Agent, an open-source framework from Nous Research. Mercury is what we built on top: a multi-domain skill stack and a six-surface gateway, designed so a single human can run a single agent across every device they own — without ever sending a token to a cloud they don't control.

"Privacy as an architectural property, not a promise."

Why it matters

Personal AI today comes in two shapes: a chat box on someone else's website (your conversation lives in their database) or a wrapper around someone else's API (every token is metered and logged). Both are useful. Neither is yours. Mercury is the third shape:

The cost case isn't subtle. An RTX 5090 runs about $2,500–3,000 street price. The equivalent cloud capacity is roughly $0.70/hour — about $504/month at 24/7 — break-even in five months even before counting the per-token fees a hosted equivalent would charge. Local AI is no longer a hobbyist trade-off. It's the cheapest deployment path that exists for an always-on personal agent.

Four specialist skills — ones we actually use

Most demo agents ship with a curated set of party tricks. Mercury ships with the four skill bundles the team needed while building Cortex (the sister project). They are not a polished demo set — they are the live skills running on the live agent.

chicago-education
Where to study in Chicago
Eight public + five private universities. MBA modalities (full-time / part-time / EMBA / online). Tuition + ROI analysis. Transfer pathways.
chicago-tax-legal
Cook County tax & tenant law
Property tax appeals. The Residential Landlord and Tenant Ordinance (RLTO). State and circuit court structure. Legal aid org directory.
threejs-design-dev
3D web graphics
React Three Fiber idioms. GLSL shader patterns. The Blender → glTF pipeline. WebXR. Performance budgets for in-browser 3D.
nous-hackathon
Submission packaging
Demo scripts. Deployment recipes. Video storyboards. Judge-checklist artifacts. (This skill wrote large parts of this page.)

Adding a fifth skill is a matter of writing a YAML manifest and dropping it into a folder. The dispatcher auto-loads it by domain context — no slash-commands, no manual routing. The hard part isn't the skills; the hard part is the gateway, the dispatcher, the memory layer, the multi-surface plumbing. Fork those, and write your own four skills.

Architecture

Six client surfaces fan into one Hermes Gateway. The gateway routes through a local Gemma 4 brain, a tool router, a context-aware skill dispatcher, and a cross-session memory store. Four skill domains compose tools out of a five-source data layer (filesystem, web search, browser MCP, Python exec, knowledge graph). The whole stack runs across three nodes — a Mac Mini in Bloomington for skill authoring, the RTX 5090 in Chicago for inference, and Google Cloud Run as managed fallback.

Cortex — the sister project Mercury orchestrates

Mercury's most visible job is driving Cortex — a multimodal brain-response analysis system built on TRIBE v2 (Meta) + Gemma 4. Upload a short video; in about six minutes, 20,484 cortical vertices light up in 3D and Gemma narrates the response at three levels of detail (general, college, clinical).

The Three.js viewer in the Cortex demo video was written in Kimi K2.6's Apr 28 sprint (commit 50423c0d). Mercury talks to Cortex through a single skill called cortex-bridge — /scan <media_file> from any of the six doors submits a clip to the Cortex API, streams progress, and displays the narration on completion.

Run it yourself

Heads up: there is no public hosted version of the live agent. Mercury is local-first by design — the AI runs on your GPU, not ours. The architecture diagrams and demo video here are real, but to actually chat with the agent you need to clone, install, and bring your own hardware.

What you need

  • An NVIDIA GPU with at least 12 GB of VRAM. Tested on RTX 5090; a 4090, 3090, or any 12 GB+ card works for the smallest model.
  • Python 3.11 or newer.
  • Ollama, to run the language models locally.
  • Roughly 30 GB of disk space for model weights.

Setup, copy-paste-able

# 1. Clone Mercury
git clone https://github.com/AlexiosBluffMara/mercury
cd mercury

# 2. Make a Python environment for it
python -m venv .venv
source .venv/bin/activate    # Windows: .venv\Scripts\activate

# 3. Install Mercury
pip install -e .

# 4. Pull the local AI models (~10 GB download)
ollama pull gemma4:e4b
ollama pull embeddinggemma:300m

# 5. Talk to it
mercury chat

That's it. Type a question. It thinks on your GPU, replies in your terminal.
