# Mercury ‚Äî working notes for Claude Code

## What this is

Mercury is a local-first AI agent for the RTX 5090 desktop, forked from
[NousResearch/hermes-agent](https://github.com/NousResearch/hermes-agent)
(MIT) and adapted for two purposes:

1. **General-purpose assistant** ‚Äî accessible via WhatsApp, Discord, email,
   and a Tailscale-only WebUI. Works on any task it has skills or tools for.
2. **Cortex collaborator** ‚Äî bridges into the Cortex package
   (`D:\cortex`) to drive `brain_scan` / `narrate` / `visualize` /
   `describe_input` pipelines, respecting Cortex's GPU scheduler so
   TRIBE v2 and Gemma 4 never coexist in VRAM.

The full architecture and roadmap will live in `PLAN.md` (TODO). For
upstream Hermes documentation, see `AGENTS.md` and the original `README.md`
section.

## Two brains

Mercury routes turns between two providers and escalates on demand:

| Brain | Model | Multiplier | Use |
|-------|-------|-----------|-----|
| Fulltime | GPT-5 mini (Copilot) | 0 | Default agent loop, planning, tool use |
| Fulltime | GPT-4o (Copilot) | 0 | Vision when Gemma 4 unavailable |
| Fulltime | GPT-4.1 (Copilot) | 0 | Backup / consistency mode |
| Part-time | Gemma 4 E4B (Ollama, `gemma4:e4b`) | local | Memory ops, narration, vision-gating, classification, fast WhatsApp/Discord replies, Cortex narration tier 0‚Äì2 |
| Tier 1 | Sonnet 4.6 (Copilot) | 1 | Hard reasoning escalation |
| Tier 1 | GPT-5.4 (Copilot) | 1 | Hard reasoning alt |
| Tier 1 | Gemini 2.5 Pro (Vertex / AI Studio via `google-genai`) | budget | Long-context / multimodal escalation |

Routing lives in `mercury/router.py` (TODO). Override slash commands:
`/brain copilot`, `/brain gemma`, `/brain auto`. No defaults to ‚â•3x
multiplier (Opus 4.5/4.6/4.7, GPT-5.5, Opus-Fast).

## Hard invariants

1. **Tailnet-only WebUI.** The web server is reached via
   `tailscale serve` and trusts `Tailscale-User-Login` /
   `Tailscale-User-Name` headers. No public exposure, no Funnel.
2. **Cortex GPU lock.** When `cortex.gpu_scheduler.GPUScheduler.state`
   is running TRIBE, the router strips Gemma E4B from candidates and
   routes to Copilot. Mercury never loads Gemma into VRAM while TRIBE
   is hot.
3. **Local-first memory.** The default memory provider is
   `holographic` (SQLite + FTS5 + HRR). The local
   `~/.mercury/memories/MEMORY.md` + `USER.md` frozen-snapshot pair
   sits in front. No SaaS memory providers in the default config.
4. **Cortex narration uses Gemma.** Cortex narration tiers 0‚Äì2 go
   through Gemma 4 E4B. Mercury's own assistant turns may use Copilot
   or paid-tier escalations freely.
5. **No `Co-Authored-By: Claude`** in commits. Match the upstream
   Hermes conventional-commit style: `fix:`, `feat:`,
   `chore(scope):`, multi-line body explaining the *why*. Run
   `git log --oneline | head` first.
6. **No attribution comments** in code (`# Generated by Claude` etc).
7. **No unprovoked docstrings.** Comment only when the *why* is
   non-obvious.

## Tailscale topology

- `mercury` hostname maps to this RTX 5090 desktop.
- iOS/Android Tailscale app gets the phone on the tailnet ‚Äî the WebUI is
  reachable from the phone browser at `https://mercury.<tailnet>.ts.net`.
- WhatsApp + Discord + email gateway runs in the same process; reachable
  from public networks via the platforms themselves (Tailscale not
  required for those).
- No auth on the WebUI itself ‚Äî tailnet ACLs gate access.

## Self-evolution roadmap

Three reference systems Mercury will integrate (post-MVP, after the
Cortex hackathon deadlines pass ‚Äî Hermes Creative May 3, Gemma 4 Good
May 18, both 2026):

1. **[NousResearch/hermes-agent-self-evolution](https://github.com/NousResearch/hermes-agent-self-evolution)** ‚Äî
   DSPy + GEPA offline pipeline that mutates `SKILL.md` files (~$2-10/run).
2. **[huggingface/ml-intern](https://github.com/huggingface/ml-intern)** ‚Äî
   300-iteration agent loop with auto-compaction (170k tokens), ToolRouter
   for HF docs/papers/datasets/repos. Best fit for Cortex Gemma fine-tune
   research.
3. **[karpathy/autoresearch](https://github.com/karpathy/autoresearch)** ‚Äî
   minimal pattern: one `program.md` + `train.py` + `prepare.py` +
   `val_bpb` eval, 5-min experiment budget. Best for "let it run overnight
   on the 5090".

All three live under `skills/self-evolution/` as wrapper skills sharing
Mercury's docker sandbox + the FTS5 trace store.
